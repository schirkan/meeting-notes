# T-206: Robustes Fehlerhandling im Sidecar (Loopback-Ausfall = Blocker, Graceful Shutdown)

## Kontext
Status: open
Priorität: high
Parent Task: T-200
Abhängigkeiten: T-200, T-201, T-203

## Goal
Fehlerhandling im Sidecar produktionsnah robust gestalten.

## Done When
- [ ] Loopback-Ausfall markiert Session als nicht startbar.
- [ ] Unbehandelte Exceptions werden in Fehlercodes übersetzt.
- [ ] Shutdown räumt Audio/Streams deterministisch auf.

## Approach
- Zentrale Error-Klassen und Exit-Code-Mapping.
- Health-Heartbeat an Main senden.
- Recovery-Strategie nur für nicht-blockierende Fehler.
