# T-207: Sidecar framework-dependent publishen und ohne SDK zur Laufzeit starten

## Kontext
Status: review
Priorität: high
Parent Task: T-200
Abhängigkeiten: T-200, T-201, T-301

## Goal
Sidecar so ausliefern, dass auf Zielsystemen mit installierter .NET Runtime kein .NET SDK benötigt wird.

## Done When
- [x] Sidecar wird per `dotnet publish` (framework-dependent, win-x64) erzeugt.
- [x] Electron Main startet das Sidecar über eine EXE (kein `dotnet run`).
- [x] Für Dev und packaged Runtime existieren robuste, getrennte Pfadauflösungen mit klarer Fehlermeldung bei fehlendem Artefakt.

## Approach
- `publish:sidecar`-Skript in npm etablieren.
- Sidecar-Pfadauflösung im Main kapseln (`dev`: `sidecar/publish/...`, `packaged`: `process.resourcesPath/...`).
- Runtime-Fehler explizit und handlungsorientiert ausgeben (Hinweis auf `npm run publish:sidecar`).

## Log
- 2026-06-17: Implementiert: `npm run publish:sidecar`, EXE-Start in `src/main/sidecar-manager.ts`, Pfadauflösung für Dev/Packaged ergänzt und `dotnet run` aus der Laufzeit entfernt.
