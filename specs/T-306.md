# T-306: Main->Renderer IPC-Streaming für Transkriptsegmente implementieren

## Kontext
Status: review
Priorität: high
Parent Task: T-300
Abhängigkeiten: T-300, T-304

## Goal
Transcript-Streaming Main -> Renderer per IPC performant umsetzen.

## Done When
- [x] Renderer erhält laufende interim/final Segmente ohne Drop.
- [x] Status- und Fehler-Events sind separat verfügbar.
- [x] Bei hoher Last bleibt UI responsiv.

## Approach
- Dedizierte IPC-Channels für transcript/status/error.
- Batching/Debounce für Interim-Spam einbauen.
- Lifecycle-Hooks beim Fensterwechsel berücksichtigen.
